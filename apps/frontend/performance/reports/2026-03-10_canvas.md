## 원인 분석

### ⚠️ 상호작용 렌더/초: 12.8/s (주의 구간 5~15/s)

마우스 이동 3.286초 동안 42회 idle 렌더가 발생해 초당 약 12.8회의 리렌더가 측정됐습니다.
`주의` 구간 내에 있지만 상한(15/s)에 근접해 있어 추이 관찰이 필요합니다.

**가능한 원인:**

1. **MobX observable 과도 구독** — XYFlow 노드/엣지 상태를 MobX store에서 관리할 경우, 마우스 이동 이벤트가 observable을 갱신하여 불필요한 리렌더를 유발할 수 있습니다.
2. **`@xyflow/react` 내부 상태 업데이트** — ReactFlow는 내부적으로 마우스 좌표를 뷰포트 상태로 관리합니다. 뷰포트 변경 시 구독 중인 컴포넌트 전체가 리렌더될 수 있습니다.
3. **이벤트 핸들러 과잉 등록** — `jsEventListeners: 703`으로 이벤트 리스너가 다수 등록돼 있습니다. 마우스 이동 시 여러 핸들러가 연쇄 실행되면서 렌더를 트리거할 가능성이 있습니다.

### ℹ️ React 커밋 횟수: 68회 (참고)

페이지 진입부터 측정 종료까지 총 68회 커밋이 발생했습니다. 캔버스 초기 로드 + 마우스 상호작용을 포함한 수치로, 현 시점에서 이상 수준은 아니지만 노드/엣지 수가 증가하면 함께 증가할 수 있습니다.

### ℹ️ 이벤트 리스너: 703개 (참고)

현재 문제 수준은 아니나, 노드가 동적으로 추가/삭제되는 워크플로우에서 `removeEventListener` 누락 시 누수로 이어질 수 있습니다.

## 해결 방법

### 상호작용 렌더/초 개선

**1. `useStore` 셀렉터 최소화**

ReactFlow의 `useStore` 훅은 선택한 상태 조각이 바뀔 때마다 리렌더를 유발합니다. 뷰포트 전체 대신 필요한 필드만 구독하세요.

```tsx
// ❌ 비추천: 뷰포트 전체 구독
const viewport = useStore((s) => s.transform);

// ✅ 추천: 필요한 값만 구독
const zoom = useStore((s) => s.transform[2]);
```

**2. MobX `reaction` 범위 좁히기**

마우스 이동처럼 빈도 높은 이벤트와 연결된 observable은 `throttle`/`debounce`로 업데이트 빈도를 제한하세요.

```ts
import { throttle } from 'lodash-es';

onMouseMove = throttle((e: MouseEvent) => {
  this.cursorPosition = { x: e.clientX, y: e.clientY };
}, 50); // 50ms 단위로 제한
```

**3. 노드 컴포넌트 `React.memo` 적용**

커스텀 노드 컴포넌트가 `memo`로 감싸져 있지 않으면 부모 리렌더 시 항상 재렌더됩니다.

```tsx
const CustomNode = React.memo(({ data }: NodeProps) => {
  return <div>{data.label}</div>;
});
```

**4. 이벤트 리스너 정리**

`useEffect` cleanup에서 리스너를 반드시 해제하세요.

```tsx
useEffect(() => {
  const handler = (e: MouseEvent) => {
    /* ... */
  };
  window.addEventListener('mousemove', handler);
  return () => window.removeEventListener('mousemove', handler); // ← 필수
}, []);
```
